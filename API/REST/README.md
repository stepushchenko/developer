### Определение REST 
REST - означает передачу репрезентативного состояния. REST — это архитектурный стиль <br>
для веб-разработки. Архитектура REST излагает рекомендации по передаче представлений <br>
ресурсов между клиентами и серверами в сети.

### Ограничения REST
**Разделение клиент-сервер:** клиенты и серверы не зависят друг от друга.  <br>
Клиентам нужно знать только URI (унифицированный идентификатор ресурса) ресурса  <br>
сервера, к которому им нужен доступ. <br>
**Без гражданства:** запрос клиента к серверу должен быть автономным.  <br>
Серверу не нужно хранить какую-либо информацию о клиенте для выполнения запроса. <br>
**Единый интерфейс:** каждый ресурс, доступный для клиентов, должен иметь  <br>
только один логический URI и обеспечивать способ получения дополнительных  <br>
данных. Должен быть общий, последовательный подход к доступу к  <br>
ресурсам и их изменению. <br>
**Кэшируемый:** для повышения производительности и масштабируемости клиент  <br>
должен иметь возможность хранить определенные ответы сервера  <br>
(например, списки элементов, которые не меняются часто). <br>
**Многоуровневая система.** Многоуровневая система разделяет API, хранилище  <br>
данных и аутентификацию запросов между тремя отдельными серверами.  <br>
Клиент обычно не знает, подключился ли он напрямую к конечному серверу  <br>
или к одному из промежуточных серверов. <br>
**Код по запросу:** это ограничение является необязательным.  <br>
При необходимости вместо ответа статическими представлениями ресурсов  <br>
в XML или JSON сервер может отправить исполняемый код».
